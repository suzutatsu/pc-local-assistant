# カスタムアクション (Custom Actions) 一覧

本プロジェクトでは `browser-use` エージェント向けに、ブラウザUI操作以外の高度な振る舞いやAPI連携を行わせるためのカスタムツール（アクション）を用意しています。これらは `tasks.yaml` のプロンプト内で指示することでAIが自律的に利用します。

## 1. ユーザーとの対話
### `ask_user(question: str)`
- **概要**: 実行を一時停止してターミナル上に質問を表示し、ユーザーからのテキスト入力を求めます。
- **主な用途**:
  - ログイン時のMFA（多要素認証）コードの入力
  - セキュリティ上、AIに任せられない手動操作の完了待ち（例: 「ログインが完了したらEnterを押して引継ぎ」）

## 2. Jira 連携
`.env` にJiraの認証情報（URL、メールアドレス、APIトークン）が設定されている場合に利用可能です。重い画面操作を省き、API経由で高速かつ確実にチケット操作を行います。

### `search_jira_issues(jql: str)`
- **概要**: JQL（Jira Query Language）に従ってチケットを検索し、一致したチケットのリスト（最大10件）を返します。
- **主な用途**: 「自分がアサインされている進行中のチケットの一覧を見る」「特定のラベルがついたタスクを列挙する」など。
- **引数例**: `project = PROJ AND status = "In Progress"`

### `add_jira_comment(issue_key: str, comment: str)`
- **概要**: 指定したJiraチケットのコメント欄に新しいコメントを追加します。
- **主な用途**: AIが完了した作業の報告、調査結果のサマリーの書き込みなど。

### `update_jira_issue_description(issue_key: str, description: str)`
- **概要**: 指定したJiraチケットの「説明 (Description)」全体を更新（上書き）します。
- **主な用途**: 古いチケットの内容整理、情報の追記更新など。

---
> **開発者向け**: 新しいカスタム機能を追加したい場合は、`main.py` 内で `@controller.action("アクション名")` デコレータを使って関数を追加定義してください。
